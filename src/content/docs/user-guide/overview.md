# User Guide Overview

Welcome to the Digest Engine User Guide. This section is designed to help you move from first login to a reliable editorial workflow without having to guess where a feature lives or what to read next.

If you are scanning the guide for the first time, start with the **Table of Contents** tab to jump around the current page. Use the **Files** tab when you want to browse the full guide and move between topics.

## Start Here

Choose the path that matches how you are using Digest Engine:

- [**Getting Started (Hosted SaaS)**](getting-started-saas.md): Best for teams using the hosted product and onboarding a project quickly.
- [**Getting Started (Self-Hosted)**](getting-started-selfhost.md): Best for teams running Digest Engine in their own environment.

## Guide Map

Once you are set up, use these pages to work through the product in a practical order:

- [**Projects and Content**](projects-and-content.md): Learn how projects are scoped, how content is organized, and how relevance scoring shapes your feed.
- [**Newsletter Intake**](newsletter-intake.md): Forward newsletters into a project, confirm trusted senders, and troubleshoot intake issues.
- [**Newsletter Drafts**](newsletter-drafts.md): Assemble, reorder, and export drafts built from curated content and promoted themes.
- [**Entities and Authority**](entities-and-authority.md): Review entity candidates and understand how authority influences ranking.
- [**Themes and Trends**](themes-and-trends.md): Follow emerging topics, review trend suggestions, and promote useful themes into drafts.
- [**Feedback and Tuning**](feedback-and-tuning.md): Use approvals, dismissals, and relevance feedback to improve future recommendations.

## Suggested Reading Paths

- **New customer**: Start with [**Getting Started (Hosted SaaS)**](getting-started-saas.md) or [**Getting Started (Self-Hosted)**](getting-started-selfhost.md), then continue to [**Projects and Content**](projects-and-content.md).
- **Editorial workflow**: Read [**Newsletter Intake**](newsletter-intake.md), [**Newsletter Drafts**](newsletter-drafts.md), and [**Feedback and Tuning**](feedback-and-tuning.md).
- **Analytics and curation**: Read [**Entities and Authority**](entities-and-authority.md) and [**Themes and Trends**](themes-and-trends.md).

Keep this page as your index. If you are not sure where to go next, pick the section that matches the job you are trying to do today and follow the linked page.
